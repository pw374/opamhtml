kill_index is a "cached" copy of monitor's kill_index. This allows the scheduler, when deciding whether to run a job, to do a fast check in the common case that the execution context's kill_index is up to date, i.e. equal to the scheduler's global_kill_index. If it's not, then the scheduler will update the kill_index for itself and monitor (and monitor's ancestors).
